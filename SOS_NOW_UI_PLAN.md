# 🚑 SOS-NOW: 실시간 응급 지도 UI/UX 상세 기획안

본 문서는 **응급실, 약국, 동물병원** 세 가지 핵심 카테고리를 중심으로, 사용자가 위급 상황에서 가장 빠르고 정확하게 정보를 파악할 수 있도록 설계된 UI/UX 가이드라인입니다.

---

## 1. 📱 화면 구성 및 레이아웃 (Layout)

### 1.1 메인 화면 (Main Map View)
*   **전체 화면 지도**: 네이버 지도를 기반으로 현재 위치 주변의 마커 표시.
*   **상단 카테고리 탭 (고정)**: 
    *   `[🚨 응급실]` `[💊 약국]` `[🐾 동물병원]`
    *   탭 클릭 시 해당 카테고리 마커만 즉시 필터링.
*   **상단 검색바**: 병원명, 지역명 검색 (둥근 모서리, 그림자 효과).
*   **우측 하단 플로팅 버튼**:
    *   `[🎯 내 위치]`: 현재 위치로 지도 중심 이동.
    *   `[📞 119]`: 즉시 119 연결 (빨간색 원형, 펄스 애니메이션).

### 1.2 하단 요약 카드 (Quick View)
*   마커 클릭 시 화면 하단 1/3 지점에 나타나는 요약 카드.
*   좌우 스와이프로 주변 병원 탐색 가능.
*   **내용**: 이름, 거리, 현재 상태(원활/혼잡/영업중), 핵심 태그(예: "소아응급 가능").

### 1.3 상세 정보 바텀 시트 (Full Detail View)
*   요약 카드 클릭 시 전체 화면으로 확장.
*   **상단**: 병원 이름, 즐겨찾기 별 아이콘, 공유하기.
*   **중단**: 실시간 상태 대시보드 (카테고리별 상이).
*   **하단**: 액션 버튼 그룹 `[전화걸기]`, `[길찾기]`, `[실시간 제보]`.

---

## 2. 🏥 카테고리별 상세 콘텐츠 (Content)

### 2.1 [🚨 응급실] - 생존 직결 정보
*   **실시간 병상 현황**: 
    *   `[가용 병상 수 / 전체 병상 수]` 표시.
    *   큰 원형 게이지 차트로 시각화 (초록/주황/빨강).
*   **중증질환 수용 여부 (태그)**:
    *   뇌출혈, 심근경색, 복부외상, 소아응급 등 가능 여부 표시.
*   **실시간 메시지 (NEMC API 9번)**:
    *   "CT 점검 중", "안과 의사 부재로 진료 불가" 등 병원측 공지사항 상단 노출.
*   **의사 현황**: 현재 근무 중인 전문의 수 (데이터 가용 시).

### 2.2 [💊 약국] - 심야/휴일 운영 정보
*   **영업 상태**: `[영업 중]` / `[영업 종료]` / `[마감 임박]` 표시.
*   **운영 시간**: 오늘 마감 시간 (예: "새벽 01:00까지 운영").
*   **서비스 태그**: #심야약국 #공공약국 #동물약품판매.
*   **실시간 확인**: "유저 3명이 30분 전 영업 중임을 확인했습니다" 문구 노출.

### 2.3 [🐾 동물병원] - 24시/응급 진료 정보
*   **운영 타입**: `[24시 운영]` / `[야간 응급 가능]` / `[일반 진료]`.
*   **진료 가능 동물**: 강아지, 고양이, 특수동물(햄스터, 파충류 등).
*   **응급 서비스**: "응급 수술 가능", "야간 할증 안내".
*   **시설 정보**: 주차 가능 여부, 입원실 유무.

### 2.4 [⚡ AED] - 자동심장충격기 위치 정보
*   **점검 상태 표시**: "60일 이내 점검 완료" 등 신뢰도 정보 노출.
*   **상세 위치 설명**: "건물 1층 로비 안내데스크 옆" 등 구체적인 위치 텍스트 제공.
*   **사용법 가이드**: 클릭 시 간단한 AED 사용법 애니메이션/이미지 노출.

### 2.5 [📚 응급처치] - 생존 가이드
*   **심폐소생술(CPR)**: 박자에 맞춘 애니메이션 가이드 제공.
*   **기도 폐쇄(하임리히법)**: 상황별(성인/영유아) 대처법.
*   **상황별 가이드**: 화상, 골절, 출혈 등 응급 상황별 1분 가이드.

## 3. ✨ SOS-NOW만의 차별화 전략 (공식 앱 보완)

*   **🚀 쾌속 로딩 (Speed)**: 공식 앱의 가장 큰 불만인 '느린 속도'를 Supabase 캐싱 기술로 해결.
*   **🎯 의료 전용 검색 (Accuracy)**: 일반 상점이 섞이지 않는 의료 데이터 전용 검색 엔진 구축.
*   **💬 실시간 메시지 강조**: 병원에서 올린 실시간 특이사항(CT 고장 등)을 최상단에 노출하여 헛걸음 방지.
*   **🤝 유저 실시간 제보**: 공공 데이터의 시차를 보완하는 유저들의 현장 상황 생중계.

---

## 4. 🎨 시각적 요소 (Visual Elements)

### 4.1 커스텀 마커 (Custom Markers)
*   **응급실**: 원형 게이지 마커. 가운데 숫자는 가용 병상 수. 테두리 색상으로 혼잡도 표현.
*   **약국**: 알약 아이콘(💊) 핀. 영업 중일 때만 밝은 초록색, 종료 시 회색.
*   **동물병원**: 발바닥 아이콘(🐾) 핀. 24시 병원은 금색 테두리 효과.

### 4.2 상태 배지 (Status Badges)
*   **원활 (Green)**: "병상 여유", "즉시 진료 가능"
*   **보통 (Orange)**: "병상 부족", "대기 발생"
*   **혼잡 (Red)**: "병상 없음", "진료 제한", "폐쇄"

---

## 5. 🔄 사용자 흐름 (User Flow)

1.  **진입**: 앱 실행 즉시 내 주변 **응급실** 지도 노출.
2.  **탐색**: 상단 탭으로 **약국**이나 **동물병원**으로 전환.
3.  **확인**: 마커의 색상과 게이지로 1차 판단 후 클릭.
4.  **결정**: 상세 시트에서 **실시간 메시지**와 **가용 병상** 확인.
5.  **실행**: `[길찾기]` 버튼으로 네이버 지도 연동 또는 `[전화걸기]`로 최종 확인.
6.  **기여**: 방문 후 `[실시간 제보]`를 통해 다른 유저에게 현장 상황 공유.

---

## 6. 🛠 기술적 구현 포인트

*   **실시간성**: Supabase Real-time을 사용하여 DB 값이 바뀌면 마커 게이지가 즉시 변함.
*   **오프라인 대응**: 네트워크 불안정 시 마지막으로 캐싱된 데이터를 보여주며 "5분 전 정보"임을 명시.
*   **접근성**: 모든 텍스트는 시스템 설정에 따라 크기 조절 가능, 색약 사용자를 위한 아이콘 구분.
